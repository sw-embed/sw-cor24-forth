Build a runnable self-test for the emitted runtime.fth dict. The
step-005 output (compiler/out/runtime-dict.s) assembles cleanly
but hasn't been EXECUTED yet — we've only verified syntactic
validity. This step proves the threaded bodies actually compute
correctly on the emulator.

Architecture:

1. Create forth-from-forth/compiler/out/runtime-selftest.s — a
   standalone .s that:

   a. Provides the 12 asm primitives the dict references:
      do_docol_far, do_exit, do_lit, do_sp_fetch, do_sp_store,
      do_rp_fetch, do_tor, do_rfrom, do_fetch, do_store, do_plus,
      do_nand. Copy these verbatim from forth-on-forthish/kernel.s.

   b. Concatenates runtime-dict.s — the emitted dict bytes from
      step 005. (Or better: build-kernel.sh emits a complete
      wrapper file that already has the scaffolding.)

   c. Defines _start that sets up RSP, DSP, then runs a threaded
      test program that:
        - Pushes 42 on DS, runs fff_cfa_DUP, expects (42 42).
          Emit "DUP42=" then the top value as decimal.
        - Pushes (1 2), runs fff_cfa_SWAP, expects (2 1).
        - Pushes (1 2), runs fff_cfa_OVER, expects (1 2 1).
        - Pushes 5, runs fff_cfa_NEGATE, expects -5.
        - Pushes (6 3), runs fff_cfa_AND, expects 2 (6&3=2).
        - Pushes (6 3), runs fff_cfa_OR, expects 7 (6|3=7).
        - Pushes (6 3), runs fff_cfa_XOR, expects 5 (6^3=5).
        - BYE to halt.

   d. Include a `.` word (print decimal) and `CR` to make output
      readable. Can hand-code these in asm or copy from
      forth-on-forthish/kernel.s.

2. Extend build-kernel.sh (or add a new script like
   test-kernel.sh) that concatenates scaffolding + runtime-dict.s
   and runs cor24-run on the result, capturing UART output.

3. Expected UART output (exact format TBD):
     DUP(42)=42
     SWAP(1,2)=2 1
     OVER(1,2)=1 2 1
     NEGATE(5)=-5
     AND(6,3)=2
     OR(6,3)=7
     XOR(6,3)=5

4. Seed a new reg-rs test (tf24a_fff_selftest) with the captured
   baseline. This is NOT replacing tf24a_fff_fib; it's a
   dict-level smoke test that runs before fib.

Deliverables:
- compiler/out/runtime-selftest.s (generated by build-kernel.sh
  or a new test-kernel.sh).
- reg-rs/tf24a_fff_selftest captured baseline.
- All 66+ existing tests still pass; new test passes.

Out of scope:
- Integrating the emitted dict into the real kernel.s (later step).
- Parsing runtime.fth (step 007 picks that up).
- Deleting do_interpret (step 009).

Why a self-test here: step 005 proved the asm assembles. This
step proves it EXECUTES correctly. Future steps assume both.
Skipping this risks compounding bugs in the dict that only
surface when fib-through-the-full-kernel fails in step 008
or 009, where debugging is much harder.
