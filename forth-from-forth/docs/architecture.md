# Architecture — forth-from-forth

## Directory layout

```
forth-from-forth/
  kernel.s              Generated by scripts/build-kernel.sh.
                        Committed so fresh clones don't have to
                        rebuild, but rebuildable at any time.
  core/
    runtime.fth         1st: :, ;, basic stack ops via SP@, NAND
                          derivatives, NEGATE.
    minimal.fth         2nd: BEGIN/UNTIL/IF/THEN/ELSE/0=/=/ (/\
    lowlevel.fth        3rd: WORD, NIP, TUCK, ROT, 2DUP, 2OVER,
                          ABS, *, /, MOD, /MOD, -, AND, OR, XOR,
                          CONSTANT/VARIABLE, DO/LOOP/?DO/I/UNLOOP.
    midlevel.fth        4th: CR SPACE HEX DECIMAL .
    highlevel.fth       5th: DEPTH .S WORDS VER PRINT-NAME '
                          >NAME SEE-CFA SEE PRIM-MARKER DUMP-ALL
                          FIND STR= NUMBER DIGIT-VALUE
                          INTERPRET QUIT + install/invoke.
    prims.fth           Hand-written PRIM: declarations bundling
                          each asm primitive's name + body.
                          Consumed by the cross-compiler when
                          emitting the asm-primitive section of
                          kernel.s.
  compiler/
    xcomp.fth           The cross-compiler itself. Pure Forth.
                        Loaded into the phase-3 forth-on-forthish
                        REPL; defines alternate-compilation words
                        (PRIM:, COLON:, ;COLON, VARIABLE:,
                        CONSTANT:, IF:, THEN:, BEGIN:, AGAIN:,
                        UNTIL:, WHILE:, REPEAT:) that EMIT COR24
                        assembler text instead of updating the
                        target dict.
    README.md           usage + design cross-reference.
  scripts/
    build-kernel.sh     Regenerate kernel.s: cat xcomp.fth +
                        core/*.fth + compile-cmd | cor24-run
                        --run forth-on-forthish/kernel.s
                        --terminal | strip-markers > kernel.s.
    demo.sh             Run an example against the generated
                        kernel.s. Same usage as the phase-3
                        demo.sh.
    see-demo.sh         SEE demo, non-interactive.
    dump.sh             DUMP-ALL every dict entry.
    repl.sh             Interactive REPL.
  docs/                 this directory.
```

## Bootstrap flow — the emitted kernel

`cor24-run --run kernel.s -u <input>` starts at `_start` in the
generated `kernel.s`:

1. Load RSP (`la r1, 983040`), snapshot initial `sp` into
   `var_sp_base`.
2. Install Forth QUIT's CFA in `var_quit_vector`. The compiler
   knew QUIT's label at emit time.
3. `jmp` to Forth QUIT's CFA. Never returns.

No UART read on boot. No asm INTERPRET. No hash-table populate.
~20 asm instructions from reset to first Forth word executing.

## Bootstrap flow — generating kernel.s

1. `scripts/build-kernel.sh` cats `compiler/xcomp.fth`,
   `core/prims.fth`, `core/runtime.fth`, `core/minimal.fth`,
   `core/lowlevel.fth`, `core/midlevel.fth`, `core/highlevel.fth`,
   plus a trailing `COMPILE-KERNEL` invocation.
2. Pipes into `cor24-run --run forth-on-forthish/kernel.s
   --terminal`. The phase-3 Forth REPL interprets the input:
   - `xcomp.fth` defines the compilation vocabulary (PRIM:,
     COLON:, etc.) in the phase-3 dict.
   - Subsequent lines (PRIM: declarations, COLON: defs,
     IMMEDIATE control-flow emitters) execute these words,
     which EMIT assembler text to UART.
   - The final `COMPILE-KERNEL` token wraps the emitted text
     with `!!BEGIN-KERNEL!!` / `!!END-KERNEL!!` markers.
3. The UART stream is piped to `sed -n
   '/^!!BEGIN-KERNEL!!$/,/^!!END-KERNEL!!$/p' | sed '1d;$d'` to
   strip REPL chrome (" ok" lines) and the markers themselves,
   yielding clean assembler source.
4. Redirect into `forth-from-forth/kernel.s`.

## What lives where

- **Asm primitives (~35 words)**: defined as PRIM: declarations
  in `core/prims.fth`. The cross-compiler emits each one as a
  `.word <link> / .byte <flags_len> / .byte <chars> / <asm body>
  / NEXT` block.
- **Forth colon defs (~55 words)**: defined in the five
  `core/*.fth` tiers with the same `:` ... `;` syntax as phase
  3. The cross-compiler processes them through remapped
  compile-mode emitters.
- **Runtime variables**: HERE, LATEST, STATE, BASE,
  var_sp_base, var_quit_vector — emitted as `.word` cells at
  the end of kernel.s. LATEST init is the label of the last
  emitted dict entry.
- **DOCOL trampolines**: near-DOCOL is not used (image too
  far from do_docol); far-DOCOL (`push r0; la r0,
  do_docol_far; jmp (r0)`) is the only form emitted.

## Changes from phase 3

| Thing | Phase 3 | Phase 4 (this) |
|---|---|---|
| kernel.s | Hand-written | Generated |
| Asm do_interpret | Lives | Not emitted |
| Asm do_word, do_find, do_number | Live | Not emitted |
| Asm do_quit_ok, do_quit_restart | Live | Not emitted |
| tick_word_cfa | Lives | Not emitted (Forth `[']` from phase 3 is the only one) |
| Hash table | 256 buckets + compute_hash primitive | No table, no compute_hash |
| _start hash populate loop | Runs | Gone |
| Bootstrap INTERPRET load of core | Reads core/*.fth from UART | Dict already in image; no UART at boot |

## What does NOT change from phase 3

- The five `core/*.fth` files (byte-identical).
- All asm primitive bodies currently in phase 3
  (`do_emit`, `do_key`, `do_plus`, `do_sp_fetch`, etc.).
- The threading model (DTC NEXT, DOCOL prelude).
- The register allocation (r0=W, r1=RSP, r2=IP).
- Forth INTERPRET/QUIT behavior. Since `highlevel.fth` already
  has them (shipped in subset 20), and they read from UART via
  Forth WORD, the emitted kernel boots straight into the same
  REPL behavior.
